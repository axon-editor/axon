## Axon v1.2.3

Axon v1.2.3 is the next local-agent hardening release after v1.2.2. It keeps
the already shipped local tool foundation, then tightens the edge cases around
conversation memory, edit proposals, local model hangs, project context limits,
workspace trust, and the first terminal companion flow.

### Axon terminal companion

- Added the first `axon` CLI build from the agent-specific source path without
  renaming the existing core server entrypoint.
- `axon .` and `axon /path/to/project` open projects through the Axon desktop
  app.
- `axon ask "prompt"` streams project-aware answers through `axon-core`.
- `axon commit` reads `git diff --staged`, streams a draft commit message, and
  asks before running `git commit`.
- `axon fix` reads the current Problems snapshot exported by Axon, streams a
  local fix request, applies edit proposals only inside the active workspace,
  and lets file watchers/LSP refresh diagnostics.
- `axon-core` now writes `~/.axon/core.port` on startup so terminal commands
  can find the running backend without guessing the port.
- Axon now exports `~/.axon/diagnostics.json` from the renderer so terminal
  agent commands can work from the same Problems data shown in the UI.
- The packaged core build now emits both `axon-core` and `axon`.
- Axon now checks whether `/usr/local/bin/axon` is installed and can prompt the
  user to install or update the command-line tool from inside the app.

### Cold start and identity

- Packaged startup now creates the editor window before waiting for bundled
  `axon-core` health, so the shell can render while backend readiness catches
  up in the background.
- The status bar now carries Axon's first persistent visual identity pass with
  an animated accent strip and stronger active state treatment for editor
  controls.
- Monaco semantic token colors now explicitly cover functions, methods,
  classes, interfaces, parameters, properties, constants, and types.
- Sidebar file and folder names now use Git state color again while preserving
  the compact dot and status-label indicators.
- The editor now allows scrolling beyond the last line for more comfortable
  navigation at the end of a file.

### Agent context and runtime hardening

- Follow-up requests through the legacy local-provider path now include recent
  conversation history instead of rebuilding every request as a stateless turn.
- Non-streaming local-provider calls now have a finite timeout so stalled local
  model requests fail instead of hanging indefinitely.
- Project context packs are trimmed to a conservative token budget before they
  are sent to local models.

### Edit proposal safety

- Edit proposal paths are normalized against the active workspace before Axon
  applies generated file changes.
- Edit proposals that point outside the workspace are rejected and reported in
  the conversation instead of being written blindly.
- Multi-file edit proposal JSON is stripped with a fence-aware approach so
  nested file objects do not leave JSON fragments in the rendered response.

### Workspace trust

- Ask Axon command entry points now respect workspace trust before opening the
  agent sidebar.
- This matters because local agent tools can read files, list workspace paths,
  and search project text through `axon-core`.
