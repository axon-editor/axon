## Axon v1.2.5

Axon v1.2.5 is a detailed editor polish release. It improves the onboarding
flow, the command-line agent input experience, theme coverage across the app,
and the sidebar file watcher behavior that previously made external folder
moves appear in Git before they appeared in the visible file tree.

### Onboarding and layout

- Added a tabbed Welcome to Axon experience so onboarding behaves like a real
  editor tab that can sit beside normal files and previews.
- Kept onboarding as a first-run experience instead of showing it every time a
  workspace changes.
- Added app-wide sidebar docking so the main sidebar can move to the right and
  stay there across workspaces.
- Improved welcome responsiveness when the sidebar is open and tightened the
  welcome layout so it fits the editor shell better.

### Terminal agent and workspace handoff

- Refined the `axon` terminal composer with a fuller input box, multiline
  entry, Escape interruption, and cleaner local-command handling.
- Hardened terminal streaming so interrupted requests report a clear stopped
  state instead of leaving the session ambiguous.
- Improved `axon .` and CLI workspace handoff so folders opened from the
  terminal route through the same workspace-loading path used by Axon's folder
  picker.
- Continued shaping the CLI toward a serious Codex/Claude Code style
  conversation loop while keeping it Axon-specific.

### Themes and editor surfaces

- Added built-in One and Solarized theme families.
- Moved built-in theme source assets into `editor/public/assets/themes` so the
  theme catalog is easier to expand.
- Removed unused React starter SVG assets from the renderer bundle.
- Made more of the editor shell theme-aware, including sidebar hover and active
  states, workspace root headers, tab surfaces, markdown preview, error
  messages, modals, and the AI sidebar.
- Fixed folder picker/open modal readability on light themes by replacing
  fixed dark palette colors with active Axon theme tokens.
- Reduced gitignored sidebar dimming so ignored files and folders remain
  readable without losing their muted state.

### File watching and sidebar refresh

- Fixed expanded sidebar folders after external disk changes. The watcher was
  firing and Git decorations could update, but expanded folders kept their
  lazy-loaded child cache until the sidebar remounted.
- Added a tree refresh nonce through the sidebar so expanded folder nodes
  refetch their children when the app shell publishes a fresh watched tree.
- Preserved lazy folder loading for fast workspace startup while making Finder
  moves and other external disk changes visible without switching folders.

### Editor controls

- Added keyboard font zoom shortcuts with command/shift plus and minus so the
  editor text size can be adjusted without opening settings.
