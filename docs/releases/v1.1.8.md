## Axon v1.1.8

Axon v1.1.8 focuses on day-to-day editor reliability. It hardens the terminal
for long AI sessions, makes Git history and source control easier to read,
adds real file drag/drop flows, introduces pinned editor tabs, and adds a
custom in-editor Find surface.

### Terminal

- Increased xterm scrollback so long-running AI and TUI sessions keep much more
  output in the terminal buffer.
- Reduced the chance of older terminal text disappearing when a session prints
  a lot of content.
- Kept the terminal reconnect path quiet so Axon does not write internal
  connection banners into the user's visible shell output.
- Kept the Shift+Enter terminal path using a richer key sequence so terminal
  apps can treat it as multiline input instead of a normal submit.
- Added clearer shortcut text to terminal tooltips.

### Editor tabs

- Added tab pinning from the editor tab context menu.
- Pinned tabs stay at the front of the pane.
- Closing other tabs and closing tabs to the right now preserve pinned tabs.
- Restored workspaces normalize older saved layouts so missing `pinnedTabs`
  data does not break session restore.
- Open tabs whose files were deleted from disk now render with a red,
  strikethrough treatment so the tab state is obvious.

### Editor Find

- Added a custom Find overlay for `Cmd+F` / `Ctrl+F` inside Monaco editors.
- Added next and previous match controls.
- Enter moves to the next match.
- Shift+Enter moves to the previous match.
- Escape closes the Find surface and returns focus to the editor.
- Find matches now get inline decorations with a stronger active-match
  highlight.

### Sidebar drag and drop

- Added external file/folder import support from Finder or Explorer into the
  sidebar.
- Dropping files into a workspace root imports them into the project.
- Dropping files or folders onto a folder imports them into that folder.
- Dropping a folder onto the empty no-workspace sidebar opens that folder as
  the workspace.
- Guarded imports against missing sources, overwrites, and copying a folder
  into itself.
- External drag detection now uses Electron's real file path extraction instead
  of relying on browser-only drag metadata.

### Git watcher and file state

- Made Git status refresh faster after file creation, modification, deletion,
  and external filesystem changes.
- The normal folder watcher now also nudges Git status updates because new
  untracked files do not always mutate watched `.git` paths immediately.
- Newly created or changed files should update sidebar Git colors more quickly.

### Source control and Git history

- Git history now fetches the full available commit history instead of stopping
  at a short fixed limit.
- Git author avatars now use GitHub noreply emails when possible.
- Gravatar lookups now avoid generated placeholder avatars so Axon can fall
  back to initials when a real avatar does not exist.
- Commit file diffs now include old and new file contents, not only raw diff
  text.
- Source Control and Git History now use Monaco's diff editor for readable
  side-by-side changes.
- Commit diffs handle renamed files by loading the old path from the parent
  commit and the new path from the selected commit.
- Loading states in Source Control and Git History now use skeleton rows instead
  of plain loading text.

### LSP and shutdown stability

- Added more guards around IPC sends so closing or quitting a window does not
  send messages into destroyed Electron webContents.
- Improved language-server failure capture so stdout/stderr is preserved before
  the process result is reported.
- Improved workspace marker detection for shallow projects so language servers
  do not depend only on markers at the root.
- Settings now avoids showing unrelated idle language servers as permanent
  loading rows when a workspace does not need them.

### UI polish

- Improved sidebar context menu positioning so menus stay inside the viewport.
- Smoothed context menu presentation and kept action modes more focused.
- Inline create rows now use Axon's file/folder icon system instead of generic
  Lucide icons.
- Workspace search history is scoped by window/workspace session instead of
  being one global list across the whole app.

### Notes

- This release keeps the committed Spotify build config empty. Release builds
  still inject the managed Spotify client ID during packaging.
- The old duplicate release-notes folder is still avoided; release notes live
  under `docs/releases`.
