## Axon v1.1.5

This release tightens day-to-day editor behavior around the terminal, platform
chrome, Source Control, and sidebar file creation.

### Fixed

- Kept terminal sessions alive when the renderer WebSocket disconnects, so a
  long-running terminal can reconnect instead of losing its process.
- Added a packaged core watchdog that restarts the bundled Axon backend if it
  stops responding after the app has been open for a long time.
- Preserved native window controls while moving Axon toolbar controls away from
  Windows titlebar buttons.
- Fixed Windows workspace labels so the titlebar folder button shows the folder
  name instead of the full absolute path.
- Prevented Source Control from sending invalid placeholder paths like
  `src/types/null` into Git diff/action commands.

### Improved

- Added inline sidebar creation for files and folders, matching the expected
  editor flow where the textbox appears at the parent folder or workspace root.
- Creating an empty inline entry now cancels on blur, while a typed name creates
  the file or folder on blur or Enter.
- Added duplicate-name feedback directly in the inline sidebar textbox.
- Added a fixed workspace-root drop/create strip and sidebar drag resizing.
- Hid workspace and file-specific toolbar/status actions until a folder or file
  is actually active.

### Verification

- `npm run build:main`
- `npm run build:renderer`
- `npm run build:core`
- `go test ./...`
- `git diff --check`
