## Axon v1.1.6

Axon v1.1.6 is a large stability and editor-foundation release. It covers the
Windows titlebar fix, terminal reliability work, the main/renderer architecture
split, Git history, language-server packaging checks, TypeScript navigation,
and Spotify/status-bar polish.

The main goal of this release is to make Axon feel less like a prototype shell
around Monaco and more like a daily editor: terminals should stay alive, LSP
startup should be less fragile, TypeScript should understand the project SDK,
Git history should live in the sidebar, and the app should package the same
language-server behavior that works during development.

### Important packaging note

- GitHub release builds now run `npm run build:language-servers` before desktop
  packaging.
- GitHub release builds now verify managed language-server bundles before
  `electron-builder` creates installers.
- This means packaged release assets should include the managed language-server
  folders expected by Axon's runtime resolver.
- Local development still uses the already-present
  `editor/build/language-servers/<platform>` folder. If that folder is missing
  during a local package build, run `npm run build:language-servers` first.

### Windows and platform chrome

- Preserved native Windows window controls instead of forcing custom controls
  across every platform.
- Moved toolbar/status controls away from the Windows close, minimize, and
  maximize buttons.
- Kept the macOS traffic-light spacing behavior while letting Windows use its
  native control area.
- Fixed the workspace folder label behavior so Windows shows the folder name
  instead of the full absolute folder path.
- Kept file/workspace-specific status actions hidden when there is no active
  folder or active file.
- Continued to use the native platform titlebar model while adjusting Axon's
  internal chrome to avoid collisions.

### Terminal reliability

- Improved the integrated terminal startup environment so Axon launches the
  user's real shell instead of a minimal GUI-app shell.
- Started zsh and bash as login interactive shells so version managers, shell
  functions, aliases, and user profile setup have a chance to load normally.
- Added a conservative PATH fallback for common developer tool locations before
  shell startup files run.
- Fixed terminal behavior where commands installed on the machine could be
  missing inside Axon even though they worked in the system terminal.
- Kept the renderer reconnect path quiet so temporary WebSocket reconnects do
  not write noisy reconnect banners into the user's terminal scrollback.
- Added WebSocket ping/pong keepalive in axon-core so idle terminal sessions do
  not detach after the app has been open for a long time.
- Added read deadlines and write deadlines to the terminal WebSocket path so
  dead sockets are detected and replaced instead of silently hanging.
- Changed PTY output transport from WebSocket text frames to binary frames.
- Fixed terminal replay byte offsets by counting real socket bytes instead of
  re-encoding terminal output strings in the renderer.
- Updated the renderer to accept `ArrayBuffer` and `Blob` terminal payloads and
  write `Uint8Array` data into xterm.
- Preserved terminal sessions across panel hide/show; hiding the terminal still
  behaves like minimizing it, not closing it.

### Language server startup and packaging

- Increased the LSP initialize timeout from a short fixed timeout to a longer
  two-minute timeout for managed servers that index real projects before
  answering initialization.
- Added bounded LSP initialize retries so a slow server can recover without
  leaving the session permanently broken.
- Cancelled initialize retries when the session is stopped, disposed, or the
  process exits.
- Added release-time language-server bundle verification with
  `editor/scripts/verify-language-servers.mjs`.
- The verifier checks the stable wrapper under `bin/` and the downloaded
  upstream runtime under `runtime/`.
- The release workflow now fails before packaging if managed servers are not
  present for the current platform.
- This makes release failures easier to diagnose than discovering missing LSP
  binaries inside a downloaded app.

### TypeScript and definition navigation

- Fixed TypeScript LSP initialization so Axon passes the workspace
  `node_modules/typescript/lib/tsserver.js` path when a project SDK exists.
- Added a bundled TypeScript SDK fallback for packaged Axon builds.
- This makes TypeScript projects more likely to resolve local declaration
  files, path aliases, framework types, generated types, and local project SDK
  behavior correctly.
- Fixed external go-to-definition routing from Monaco into Axon's tab system.
- Cmd-click and F12 can now open a definition target from another file as a
  real Axon tab.
- Definition navigation now reuses Axon's existing navigation target flow so
  the destination line and column are revealed after the file model mounts.
- Same-file Monaco navigation remains untouched; Axon only intercepts file URI
  open requests that need the app's pane/tab model.

### Git history

- Added a full Git history view in the sidebar.
- Git history now lists commits with subject, author, relative date, short hash,
  and changed-file count.
- Selecting a commit slides the commit list out and slides the commit detail
  view in, keeping the history experience inside the sidebar rather than a tab.
- Commit detail shows author information, body text, date, hash, and changed
  files.
- Clicking a changed file opens a full editor history diff view.
- Git history diff views show the selected commit file changes in the editor
  area rather than squeezing the comparison into a small modal.
- Added Git author avatar support based on commit author email.
- Added avatar fallback initials so Git history remains readable if the remote
  image cannot load.
- Kept Git history under the sidebar view model so Files, History, and Spotify
  are treated as sidebar sections.

### Source Control and Git safety

- Continued hardening Source Control against invalid placeholder paths.
- Prevented paths containing placeholder segments such as `null` from reaching
  Git commands.
- Kept noisy Git errors like invalid placeholder path access out of the Source
  Control UI.
- Preserved Git status decoration behavior for changed files and parent folders.
- Kept ignored-path styling constrained so ignored files do not look like normal
  changed files.

### Sidebar and status bar

- Moved Files, Git History, and Spotify view switches into the bottom status
  bar beside the music area.
- Removed the duplicate Files/Git History launcher row from the bottom of the
  sidebar.
- Added separators to the status bar so workspace search, sidebar view
  switches, source control, editor state, problems/output, and terminal controls
  read as distinct groups.
- Clicking a status-bar sidebar view reopens the sidebar automatically if it
  was hidden.
- Kept the sidebar content area focused on the active section instead of using
  vertical space for repeated navigation buttons.
- Kept the resizable sidebar behavior from the previous release.
- Kept the fixed workspace-root row for root-level create, context menu, and
  drag/drop operations.

### Spotify

- Added Spotify as a sidebar view that shares app-level state with the floating
  player.
- Added a Spotify floating now-playing player that can stay visible outside the
  sidebar.
- Persisted the floating player position in local storage.
- Fixed Spotify client ID persistence so the main process reads the real user
  settings file instead of an empty path fallback.
- Saving the Spotify client ID now refreshes Spotify status immediately.
- After reconnecting, the saved client ID should be reused after quitting and
  reopening Axon.
- Playlist loading should work from the saved settings without re-entering the
  client ID every session.
- Added status handling so the Spotify panel can transition from setup to
  connected state after OAuth callback.
- Added remote artwork support for Spotify playlists and now-playing art.
- Added fallback artwork icons for playlist images, now-playing images, and the
  floating player when remote images fail.
- Allowed HTTPS images in the renderer Content Security Policy so Spotify
  artwork can load in packaged builds.

### Architecture refactor

- Split the main process out of the large `src/main/index.ts` file into feature
  folders.
- Added main-process folders for app handlers, core process management,
  diagnostics, extensions, fonts, filesystem handlers/watchers, Git, HTML
  preview, LSP, settings, Spotify, tasks, updates, and window creation/menu.
- Moved renderer code into feature folders for diagnostics, editor, extensions,
  Git, LSP, preview, search, settings, sidebar, Spotify, tasks, terminal, and
  updates.
- Moved shared renderer components and utilities into `renderer/shared`.
- Kept existing behavior while moving code to the new architecture instead of
  rewriting unrelated logic.
- Added shared contracts and bundled editor service wiring so the feature
  folders can communicate through the existing IPC/preload boundary.
- Removed the private architecture planning note from the tracked repository.
- Added `architectureNotes.md` to `.gitignore` so local planning notes stay
  local.

### Core backend and IPC

- Continued separating the Go backend process management from the Electron app
  entry file.
- Kept backend health and process lifecycle responsibilities under the main
  process core module.
- Kept renderer-facing APIs routed through preload IPC instead of raw renderer
  Node access.
- Preserved the existing app security boundary while moving feature logic into
  smaller files.

### HTML preview and renderer assets

- Kept the HTML preview feature under its own main-process and renderer feature
  folders.
- Preserved local dev server and preview injection behavior while moving files
  into the new architecture.
- Updated renderer CSP image handling to support external HTTPS image sources
  where the UI intentionally displays remote artwork or avatars.

### Settings and persistence

- Kept Spotify client ID in Axon's user settings.
- Continued routing settings updates through the main-process settings handler.
- Settings updates still normalize values before writing them to disk.
- Python LSP settings still notify active Python language-server sessions when
  interpreter or virtual environment settings change.
- Workspace settings still live in `axon.json` only when explicitly used.

### Release workflow

- Removed the duplicate `release-notes/` folder so release documentation lives
  under `docs/releases/`.
- Added managed language-server verification to `.github/workflows/release.yml`.
- Kept release notes tied to `docs/releases/<tag>.md`, which is what the GitHub
  release workflow reads.
- Kept draft release behavior for generated GitHub releases.

### Developer experience

- Kept `npm run build:main` passing after the architecture split.
- Kept `npm run build:renderer` passing after the renderer feature move.
- Kept `git diff --check` clean after the refactor and follow-up fixes.
- Kept Go tests passing for the core backend after terminal WebSocket changes.

### Known notes

- Packaged LSP behavior depends on the release workflow or local developer
  running `npm run build:language-servers` before packaging.
- Some managed language servers still require system runtimes even when the
  server launcher is bundled. Java/Kotlin need a JDK and C# needs .NET for full
  project analysis.
- TypeScript now points to the workspace SDK when present, but project-specific
  configuration still depends on valid `tsconfig.json`/`jsconfig.json` and
  installed project dependencies.
- Spotify playback control still depends on Spotify Premium and an active
  Spotify device, matching Spotify Web API requirements.
