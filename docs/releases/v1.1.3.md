# Axon v1.1.3

This release fixes the Python LSP experience and adds project virtual
environment selection so Axon can resolve imports from the same runtime your
Python project uses.

## Added

- Added Python virtual environment selection in Settings.
- Added interpreter detection for selected Python environments.
- Added Pyright configuration that points at the selected interpreter and venv.
- Added a Language Servers restart action for reloading LSP runtime changes.
- Added immediate Python built-in completions for common names such as `print`
  while Pyright is still starting.
- Added LSP startup and runtime logs to the Output panel.
- Added clearer Language Server Settings statuses for bundled, running,
  missing, and failed-start servers.
- Added runtime requirement messages for Python, Java, Kotlin, and C#.

## Changed

- Completion requests now auto-start the matching language server when possible
  and wait briefly for initialization before returning an empty result.
- Python server status now shows the interpreter being used, or explains that no
  Python virtual environment is selected.
- Settings can now open the Output panel directly from the Language Servers
  section so LSP logs are one click away.

## Fixed

- Fixed Python autocomplete feeling empty on first use when Pyright had not
  finished starting.
- Fixed Python project imports such as Django/DRF packages not being resolvable
  when they live inside a project virtual environment.
