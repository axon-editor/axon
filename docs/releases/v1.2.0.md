## Axon v1.2.0

Axon v1.2.0 continues the editor hardening work from the previous release and
captures the remaining feature batch that was still sitting in the working tree.
This update sharpens the file tree, improves Git status handling, makes font
previewing easier, and keeps workspace trust changes explicit.

### Sidebar and file tree

- Sidebar file operations now stay in sync locally when files or folders are
  created, renamed, moved, or deleted.
- Root-level context actions are available again when a workspace has content,
  which makes it easier to create new entries without hunting for empty space.
- Tree operations are tracked immediately so the UI updates before the next
  refresh cycle finishes.

### Git status

- Git status parsing now uses NUL-delimited output, which keeps paths with
  spaces, screenshots, and other media filenames from being misread.
- File-name status colors now apply consistently to media paths instead of
  leaving them visually stale.
- The tree decoration pipeline keeps renamed and moved entries aligned with the
  real filesystem path instead of waiting for a full rescan.

### Search and font previews

- Search dropdown entries can preview their own font family.
- This makes font selection easier to verify at a glance, especially for
  imported or bundled NerdFont variants.

### Workspace trust

- Workspace trust revocation now uses a custom Axon modal instead of the native
  browser-style confirm dialog.
- The trust warning explains exactly which execution surfaces are disabled when
  a workspace is marked untrusted.

### Cursor settings

- Cursor blinking settings now call out the available modes explicitly,
  including `expand`.
- The cursor controls remain tied to Monaco so the saved editor preference is
  reflected immediately in the active editor.

### Notes

- This release note covers the remaining feature work that was not part of the
  previous pushed release note.
- The untracked screenshot used while working is intentionally left out of the
  release.
