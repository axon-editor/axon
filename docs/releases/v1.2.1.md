## Axon v1.2.1

Axon v1.2.1 is the local editor workflow release. It brings the big v1.2
feature batch into the release docs: fuller Git workflows, multi-root workspace
foundations, richer language tools, and a native Test Explorer.

### Full Git workflows

- Added local branch creation and checkout controls.
- Added stash creation with untracked-file support.
- Added stash apply, pop, and drop actions.
- Added merge conflict listing and conflict resolution helpers.
- Added worktree listing, creation, removal, and pruning controls.
- Added a compact commit graph preview.
- Fixed stash listing by reading a stable formatted Git output, so entries such
  as `WIP on main` show up correctly in Axon's Git workflow panel.
- Clarified the stash icon tooltip so it explains that stashing saves
  uncommitted changes and hides them from Source Control.

### Multi-root workspaces

- Added workspace overview foundations for richer project/window setups.
- Added root switching from the workspace overview surface.
- Added workspace overview summaries for diagnostics, tests, Git state, and
  root status.
- Kept the implementation local-only and avoided cloud/API workflow coupling.

### Advanced language features

- Added a Language Tools modal for definition, references, rename, formatting,
  server status, and file symbols.
- Improved LSP completion normalization for insert/replace edits, defaults,
  docs, commit characters, and additional text edits.
- Added file-symbol/buffer-symbol navigation from the editor breadcrumb path.
- Fixed `.tsx` editor handling by keeping TSX files on Monaco's supported
  `typescript` language id while using the `.tsx` path for React-aware
  completion behavior.
- Filtered duplicate Monaco TypeScript worker markers from Problems so project
  LSP diagnostics stay the source of truth.

### Testing UI

- Added a Test Explorer modal.
- Added test provider discovery.
- Added individual test target rows and per-target run actions.
- Added test output streaming and run result feedback.
- Wired test discovery and run actions through the local Electron/main-process
  bridge.

### Cmd+P project search

- Cmd+P now builds a project file index instead of only searching the lazy
  sidebar tree.
- Files inside folders that were never expanded in the sidebar can now appear
  in Cmd+P results.
- Normal Cmd+P search is file-first.
- Typing `>` switches the palette into command search.
- File matching now works better across nested paths and separators.

### Project indexing

- Dependency, build, and cache folders are skipped across common project types
  to keep Cmd+P fast.
- The skip list covers common JavaScript, Python, Rust, Go, Java, .NET, mobile,
  native, and Zig output folders.
- Go cache folders such as `.gocache`, `.gochache`, and `.go-build` are now
  excluded.

### Sidebar Git states

- File tree Git colors no longer tint the whole row.
- Git state now appears as a small colored dot and short label beside the file
  or folder.
- Deleted state wins over added state when files move through create/delete
  transitions, which avoids confusing green/deep-red/deep-yellow states.
