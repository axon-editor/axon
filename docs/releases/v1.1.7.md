## Axon v1.1.7

Axon v1.1.7 is a focused stability release for the work after v1.1.6. It fixes
the terminal behavior, makes Spotify feel built into Axon
instead of user-configured, adds the first Axon Agent sidebar surface, and
hardens packaged language-server resolution so production builds use the same
bundled servers that work in development.

### Terminal

- Preserved the terminal scroll position when switching between terminal tabs.
- Avoided fitting hidden xterm instances during tab switches so long terminal
  output does not jump back to the top.
- Fixed the visual prompt wrapping glitch that could show a duplicated prompt
  after switching terminal tabs.
- Added terminal keyboard handling for Shift+Enter so multiline shell input can
  move to the next line instead of immediately submitting.
- Kept terminal reconnect state out of the terminal buffer so Axon does not
  write internal connection messages into the user's shell history area.
- Routed confirmed terminal links through Axon's external-link handler so they
  open in the default browser after the user accepts the confirmation dialog.

### Windows and window lifecycle

- Fixed the New Window menu action after the main-process architecture split.
- New Window now creates a fresh managed Axon window instead of calling a stale
  no-op menu callback.
- Window cleanup still removes session restore state when a window closes.
- macOS activate behavior now uses the same managed window creation path.
- Disabled electron-updater's Windows NSIS signature gate for Axon's current
  unsigned personal builds so a downloaded update can install instead of being
  rejected after download.
- Windows SmartScreen may still warn on first install until Axon has a real
  code-signing certificate; this fix targets Axon's in-app update flow, not
  Microsoft's reputation warning.

### Spotify

- Moved Spotify setup toward Axon's built-in client flow.
- Added build-time Spotify client ID generation so release builds can ship with
  Axon's Spotify client ID while local development can still use `.env`.
- Kept the committed fallback empty so local secrets are not stored in source.
- Fixed the build config writer so an intentionally empty environment override
  is respected.
- Removed the manual Spotify client ID input from the user-facing setup flow.
- Updated the Spotify setup copy so it feels like an Axon feature instead of a
  missing configuration screen.
- Added Spotify device discovery.
- Added a device selector so users can choose the active Spotify device before
  playback.
- Removed the placeholder device option from the selector list; only real
  devices appear as selectable options.
- Added playlist infinite scroll so Axon can load beyond Spotify's first page
  of playlist tracks.
- Added loaded/total track count feedback while browsing a playlist.
- Shared selected Spotify device state between the sidebar panel and floating
  player.
- Playback now sends the selected Spotify device ID when available.

### Axon Agent sidebar

- Added the first right-sidebar surface for Axon Agent.
- The Agent toggle appears in the status bar only when AI is enabled in
  settings.
- The Agent sidebar stays hidden when AI is disabled, making settings the
  source of truth for the toggle.
- Added a dim Axon logo treatment that matches the empty editor surface.
- Added a typewriter message cycle for the Agent preview.
- The Agent message writes itself, pauses for three seconds, deletes, and moves
  to the next message instead of stacking multiple phrases.

### Search select and UI polish

- Updated the reusable search select so it can show a placeholder when no value
  is selected instead of defaulting to the first option.
- This keeps selectors like Spotify devices from making a fake first option
  look selected.
- Removed the extra sidebar open-folder footer action so the sidebar stays
  focused on project content.

### Language-server packaging

- Checked the other bundled language servers after the packaged Python/Pyright
  failure.
- Added a shared bundled-app path resolver for files that may live under
  `app.asar.unpacked` in packaged Electron builds.
- Python/Pyright now resolves from the unpacked packaged app path when Axon is
  installed.
- TypeScript language server now resolves from the same bundled-app path helper.
- TypeScript `tsserver.js` fallback now resolves from the unpacked packaged
  app path.
- Docker language server now resolves from the same packaged path helper.
- Tailwind CSS language server now resolves from the same packaged path helper.
- Project TypeScript diagnostics now use the bundled-app resolver for the
  fallback `tsc.js` path.
- Expanded electron-builder `files` and `asarUnpack` coverage for all npm-backed
  language-server packages Axon advertises:
  - TypeScript
  - Tailwind CSS
  - Docker
  - Python/Pyright
  - PHP/Intelephense
  - HTML/CSS/JSON
  - YAML
  - Bash
  - Svelte
  - Vue
  - Astro
  - GraphQL
  - MDX
  - Prisma
- Managed native/runtime-backed bundles for Go, Rust, C/C++, Java, C#, Kotlin,
  and Lua continue to ship through `extraResources/language-servers`.

### Packaging notes

- Release builds should inject `SPOTIFY_CLIENT_ID` before `build:main` so Axon's
  Spotify client ID is compiled into the packaged app.
- Local builds without `SPOTIFY_CLIENT_ID` still compile and show the friendly
  Spotify setup fallback.
- The release workflow no longer places explanatory comments inside the asset
  glob block, so the release action only evaluates real file patterns.
- The broader LSP package inclusion increases packaged assets, but it keeps
  production behavior closer to development and avoids missing runtime files
  from npm-backed language servers.
