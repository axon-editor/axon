## Axon v1.1.9

Axon v1.1.9 is a hotfix release focused on keeping the packaged app aligned
with the behavior already working in dev.

### Terminal

- Fixed terminal output replay so long-running sessions do not lose visible
  text when xterm is still catching up.
- Kept multiline terminal input on a bracketed-paste path so `Shift+Enter`
  behaves like a real newline instead of a premature submit.
- Increased the core terminal replay window and websocket write timeout so
  chatty shells and AI prompts stay stable for longer sessions.
- Increased the core PTY replay window again and delayed renderer reconnects
  until xterm has rendered queued output, preventing long AI responses from
  losing visible text during reconnect recovery.
- Ignored stale websocket messages after a reconnect so old socket data cannot
  corrupt the active terminal view.
- Kept reconnect recovery quiet so Axon does not write backend connection noise
  into the user's terminal buffer.

### Packaged LSP

- Verified the packaged app can launch Node-backed language servers from
  `app.asar` instead of depending on unpacked duplicate trees.
- Added an explicit runtime dependency for GraphQL so GraphQL language-server
  startup works in the packaged app.
- Patched the MDX markdown language-server dependency so packaged Electron Node
  can load `vscode-uri` correctly.
- Kept the release verifier checking the full language-server dependency
  closure before packaging, which catches missing runtime modules earlier.
- Added a dev-time managed LSP bundle check so a fresh clone explains when
  `npm run build:language-servers` is needed instead of silently falling back to
  missing PATH binaries.
- Made cold LSP completion requests wait briefly for server initialization so
  the first Go/Rust/Python popup does not look empty while the server warms up.
- Added `workspace/configuration` responses for Pyright so selected Python
  virtual environments flow through the same request path used by mature editor
  clients.
- Responded to basic server-side LSP requests such as workspace folders,
  progress creation, and dynamic registration so servers do not exit because
  Axon ignored their request.

### Python Environments

- Python virtual environment selection now exposes `pythonPath`, `venvPath`,
  `venv`, and common `python.analysis` options to Pyright.
- This makes imports from project environments such as Django, DRF, and other
  installed packages resolve more reliably in packaged Axon.

### Notes

- This release keeps the release process safe for dev and packaged Axon by
  focusing on production-only failures.
- The bundled language-server set is still intentionally large because Axon
  now ships the editor languages out of the box instead of asking users to
  install them one by one.
