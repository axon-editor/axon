## Axon v1.1.2

This release focuses on managed language-server packaging so Axon can ship more
of the editor intelligence it needs without requiring every project to install
the same servers manually.

### Added

- Added broader active LSP routing for Go, Rust, Python, C/C++, Java, C#,
  Kotlin, PHP, Lua, Docker, Tailwind CSS, and TypeScript/JavaScript.
- Added managed language-server bundling for Go `gopls`, Rust
  `rust-analyzer`, C/C++ `clangd`, Java `jdtls`, C# OmniSharp, Kotlin
  `kotlin-language-server`, and Lua `lua-language-server`.
- Added bundled npm-backed language servers for Python `pyright`, PHP
  `intelephense`, Docker, Tailwind CSS, and TypeScript/JavaScript.
- Expanded file and folder icon mappings for image, photo, screenshot, SVG,
  video, and broad media asset paths.

### Changed

- GitHub release builds now create platform-specific managed server bundles
  before Electron packaging starts.
- macOS release jobs now use explicit Intel and Apple Silicon runner labels so
  each DMG gets the correct `darwin-x64` or `darwin-arm64` managed server
  bundle.
- Release documentation now explains that GitHub release assets already contain
  their matching managed language servers. Source checkouts still recreate
  generated binaries locally with `npm run build:language-servers`.

### Fixed

- Fixed Linux language-server bundling by treating Open VSX `.vsix` downloads
  as zip archives instead of tar archives.
- Fixed Windows C# language-server bundling by detecting `OmniSharp.exe` inside
  the downloaded OmniSharp archive.
